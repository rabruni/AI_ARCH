# /docs

Documentation, design specifications, and architectural decision records.

## What Belongs Here

- Architecture documents and design specs
- Architecture Decision Records (ADRs) in `/docs/decisions/`
- API documentation
- System diagrams
- Runbooks and operational guides

## What Does NOT Belong Here

- Code (belongs in `/src`)
- LLM prompts (belongs in `/prompts`)
- Configuration files (belongs in `/config`)
- Generated documentation (should be in `/build` or separate docs site)
